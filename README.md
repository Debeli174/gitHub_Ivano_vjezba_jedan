# gitHub_Ivano_vjezba_jedan
PITANJA
1.	Što je Visual Studio Code i za što se najčešće koristi? 
2.	Kako se zove panel u kojem vidiš sve datoteke i mape svog projekta? 
3.	Kako možeš instalirati ekstenzije u VS Codeu i čemu one služe? 
4.	Kako otvoriti postojeći projekt ili mapu u VS Code-u? 
5.	Što se sve može raditi u Exploreru? 
6.	Kako možeš brzo pronaći određenu datoteku u projektu? 
7.	Kako možeš istovremeno gledati dvije različite datoteke u editoru? 
8.	Kako pokrećeš automatsko formatiranje koda u VS Code-u? 
9.	Što je version control i zašto se koristi? 
10.	Kako se u VS Code-u otvara panel za Git (Source Control)? 
11.	Što znače commit, push, pull? 
12.	Čemu služi commit poruka i kako treba izgledati? 
13.	Kako možeš vidjeti koje si promjene napravio u pojedinoj datoteci? 
14.	Kreiraš novu datoteku. Koji su koraci da se promjena pošalje na GitHub? 
15.	Kako se vraćaš na stariju verziju datoteke ako si pogriješio? 
16.	Na koji način možeš provjeriti status projekta i koje datoteke su izmijenjene? 
ODGOVORI
1. Visual studio code je uređivač koda koji je napravio Microsoft
2. Zove se repozitorij
3. Ekstenzije se mogu instalirati na visual studio marketplace-u i služe za dodavanje programskih jezika, alata i ostalih korisnih stvari
4. Postojeći projekt ili mapu u VS Code-u otvaramo preko File > Open Folder ili prečacem Ctrl+K, Ctrl+O.
5. U Exploreru možeš pregledavati, otvarati, uređivati i organizirati datoteke projekta.
6. Datoteku brzo nalaziš pomoću Ctrl+P i upisivanja njenog imena.
7. Dvije datoteke istovremeno gledaš pomoću Split Editor ( ikonica ili Ctrl+\ ).
8. Automatsko formatiranje pokrećeš s Shift+Alt+F.
9. Version control služi za praćenje i upravljanje promjenama u projektu.
10. Git panel u VS Code-u otvaraš klikom na Source Control ikonu ili Ctrl+Shift+G.
11. Commit sprema promjene lokalno, push šalje na server, a pull preuzima nove promjene.
12. Commit poruka opisuje što je promijenjeno i treba biti kratka i jasna.
13. Promjene vidiš klikom na datoteku u Source Control panelu.
14. Datoteku šalješ na GitHub tako da napraviš commit i zatim push.
15. Na stariju verziju vraćaš se putem Git History ili naredbom Undo/Checkout.
16. Status projekta provjeravaš naredbom git status ili u Source Control panelu.
